    How to scrape a website politely

    Polite scraping comes down to three habits: tell the site who you are with a real User-Agent, leave a gap between requests instead of hammering the server, and honor the paths the site blocks in its robots.txt. polite_bot reads your scraper's source and checks all three before you run it, so you catch the missing header or the back-to-back request loop on your screen instead of in a 429 or an IP ban. Paste a Python requests loop or a JavaScript fetch above and you get a red, yellow, or green verdict in about a second, with the exact line to fix. Nothing you paste leaves your browser: the analysis runs in a Rust engine compiled to WebAssembly.

    Identify yourself

    A request with no User-Agent, or the default python-requests/2.x string, is the first thing a rate limiter flags and the hardest for a site owner to trace back to you before they block your IP. polite_bot points at the request line and hands you a one-click header to paste back into the call.

    Space out your requests

    Back-to-back requests are the fastest way to trip a WAF or earn a 429. polite_bot looks for time.sleep, asyncio.sleep, setTimeout, and backoff libraries like tenacity, and flags the loop that fires with none of them.

    Respect robots.txt

    Paste the target's robots.txt and a path into the cross-check panel. The built-in parser resolves User-agent groups and Allow/Disallow longest-match precedence, so it reads the file the way a real crawler does, not with a guessy regex.

    Frequently asked questions

    How do you scrape a website politely?
    Set a descriptive User-Agent with a contact URL, add a delay or backoff between requests (start near one request per second and adjust to the site's Crawl-delay), and check the site's robots.txt before you request a path. polite_bot checks your code for the first two and cross-checks robots.txt for the third.
    What are web scraping rate limit best practices?
    Send one request at a time with a pause between them rather than firing a whole loop at once, honor any Crawl-delay the site publishes, back off (ideally exponentially) when you get a 429 or 503, and cache responses so you never re-request the same page. polite_bot flags a request loop with no sleep or backoff call so you fix the worst offender first.
    Is this a robots.txt checker?
    Yes, and then some. The robots.txt panel is a real parser: it groups User-agent blocks and applies Allow/Disallow longest-match precedence, so it agrees with how Googlebot and other crawlers actually read the file. It also reads your scraper's own code, which a plain robots.txt checker never does.
    Does my code get uploaded anywhere?
    No. The whole analyzer is a Rust module compiled to WebAssembly that runs inside your browser tab. There is no backend and no network request, so pasting a scraper with internal URLs or API keys is safe.
    Which languages does it recognize?
    Python (requests, urllib) and JavaScript (fetch) call shapes today. Source it doesn't recognize returns a yellow "check manually" verdict instead of a false red.
